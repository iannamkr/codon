# 논의 006: 하위 Gene + 플라스미드 시스템

## 배경

속성 시스템(005) 확정 후, 하위 Gene 설계로 넘어감. PoE 스킬 젬 레퍼런스를 참고하면서 진행.

사용자:
> "응 진행하자. 하위 gene을 설계하기 전에 우리가 조사했던 reference 문서를 참고하면서 진행하자"

## 하위 Gene 설계 후보 3개

### 방안 1: Gene에 속성 변종 부여 (16종 Gene)

```
A → A(Fire), A(Water), A(Earth), A(Plant)
T → T(Fire), T(Water), T(Earth), T(Plant)
...
= 16종 Gene
```

코돈 내 3개 Gene의 속성 조합으로 코돈 속성 결정 (순혈/우성/키메라).

- 장점: 기존 64개 코돈 구조 유지. 교배 시 속성 자연 유전. 생물학 테마 일치
- 단점: Gene의 "역할 내 다양성"이 없음. A(Fire)와 A(Water)의 차이가 속성뿐

### 방안 2: Gene에 역할 특화 + 속성 (48종 Gene)

```
A(파괴) → A-타격, A-침투, A-약화 × 4속성
T(생존) → T-보호, T-회복, T-인내 × 4속성
...
= 48종 Gene
```

- 장점: Gene 자체에 깊이. PoE 스킬 젬 다양성에 가장 가까움
- 단점: 48종은 너무 많음. 코돈 조합 110,592가지. 지나치게 복잡

### 방안 3: Gene는 단순, 코돈에서 속성 부여

Gene = A/T/G/C 4종 유지. 코돈 생성 시 별도로 속성 부여.

- 장점: 가장 단순
- 단점: "하위 Gene" 개념이 사실상 없음

## PoE2 보조젬 구조 인사이트

사용자:
> "poe2에는 스킬잼이 있고 그 스킬잼에 보조잼을 끼워넣는 방식이야. 스킬잼에 어떤 보조잼을 끼우느냐에 따라 속성이 바뀌거나 혼합되는 개념이야"

→ PoE2에서는 스킬 젬(기본 스킬)과 보조 젬(변형)이 분리됨. 속성은 스킬 자체가 아니라 보조로 변경/혼합.

이를 우리 시스템에 대입:

```
코돈 (ATG = 기폭) = 기본 스킬 (순수 효과, 속성 없음)
+ [별도 시스템] = 속성/변형 부여
→ 같은 "기폭"이라도 Fire 기폭, Water 기폭, 관통 기폭 등
```

## 코돈 + 보조 시스템 방향

### 방안 A: 하위 Gene이 보조 역할 (Gene 자체에 속성 내재)

Gene = A(Fire), A(Water) 등. 코돈 조합 시 속성이 자연스럽게 결정됨.

- 장점: 시스템 통합. 교배 시 속성 유전 자연스러움
- 단점: PoE처럼 "같은 스킬에 다른 보조를 끼워보는" 실험 재미가 약함

### 방안 B: 별도 레이어가 보조 역할 (코돈과 속성 분리)

코돈 = 순수 효과. 별도 시스템으로 속성/변형 부여.

- 장점: PoE처럼 "같은 코돈에 다른 보조를 끼워보는" 실험 재미. 빌드 레이어 명확 분리
- 단점: 시스템 2개. 교배 시 별도 설계 필요

### 방안 C: 혼합 (Gene에 속성 + 촉매로 추가 변형)

사용자:
> "B가 좋은데 갑자기 촉매는 또 뭐야"

→ "촉매"라는 이름은 아직 미확정이었음. Layer 2의 정체를 먼저 정해야 함.

## Layer 2 이름/정체 후보

### A. 효소 (Enzyme)
- 생물학: 화학 반응을 촉진/변형하는 단백질
- 느낌: "효소를 투여한다"

### B. 플라스미드 (Plasmid)
- 생물학: 세균에 외부 DNA를 삽입하는 작은 고리형 DNA. 유전공학의 핵심 도구
- 느낌: "코돈에 유전 정보를 끼워넣는다" — PoE 보조젬과 일치

### C. 전사인자 (Transcription Factor)
- 생물학: 유전자 발현을 제어하는 단백질
- 느낌: 코돈 자체는 안 바뀌고 발현 방식만 바뀜

## 결정

사용자: **방안 B (별도 레이어)** + **플라스미드** 선택.

> "ㅇㅇ B로 하자 머리터질거같아"

확정 (1차):
1. **코돈 = 순수 효과** (아미노산에 의한 효과만, 속성 없음)
2. **Layer 2 = 플라스미드 (Plasmid)** — 코돈에 삽입해서 속성/변형을 부여하는 별도 시스템. PoE2의 보조 젬에 대응

## 하위 Gene과 플라스미드의 관계 정정

사용자:
> "뭔소리야 하위 gene은 반드시 있어야하고 거기에 플라스미드가 추가되는거지"

→ 하위 Gene은 반드시 존재. 플라스미드와 하위 Gene은 별개.
- **하위 Gene**: 각 ATGC 역할 내의 분기 (코돈에 보너스 효과 추가)
- **플라스미드**: 속성/추가 변형 부여 (별도 시스템)

하위 Gene과 아미노산의 관계:

사용자:
> "아미노산은 동일, 그거까지 바뀌면 너무 복잡해"

→ 아미노산 효과는 코돈 글자(ATG, GCA 등)로만 결정. 하위 Gene은 보너스를 추가할 뿐.

## 하위 Gene 종류 설계

초안 카테고리당 3종 → 사용자 "너무 적어" → 6종 → 사용자 "더 늘려봐"

사용자:
> "맞는데 너무 적어. 우리가 수집한 poe 레퍼런스 문서를 보고 갯수를 더 확장해"

PoE1/PoE2 스킬젬 888개 + 보조젬 527개에서 메카닉 아키타입을 추출하여 카테고리당 10종으로 확장.
