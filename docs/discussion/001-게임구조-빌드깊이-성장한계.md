# 논의 001: 게임 구조, 빌드 깊이, 성장 한계

## 발단 — 빌드 깊이 문제 제기

전투 시스템 후보 3개(슬롯 순차/코돈 덱/DNA 시퀀스)를 논의하던 중, 사용자가 근본적 문제를 제기:

> "여기서 갑자기 든 생각은 poe는 기본적으로 rpg 게임이고 스킬에다가 스킬트리가 융합되서 빌드가 완성이되. 근데 우리는 gene 3개로 구성된 코돈을 나열해서 하게 되면 빌드의 맛이 사라져. 무슨 의미인지 이해했어? poe는 스킬에 잼을 넣고 그거에 맞게 스킬트리를 빌드하고 그 다음에 전투를 해보고 다시 마을에 와서 빌드를 깎고 다시 전투를 하면서 내가 만들고 원하는 빌드를 찾아가는 재미가 있는거야"

### 핵심 문제 분석

PoE의 빌드 = 다축(多軸)의 곱:
- 스킬 젬 (뭘 하느냐) × 서포트 젬 (어떻게 하느냐) × 패시브 트리 (빌드 방향) × 장비 (스탯+특수효과) = 빌드
- 이 4축이 서로 시너지/상충하면서 "내 빌드를 찾아가는" 반복 루프 생성
- 장착 → 전투 → 마을 복귀 → 조정 → 전투 → ...

현재 우리 = 1축만 존재:
- 코돈 배열 = 빌드의 전부
- 코돈을 나열하는 것만으로는 "빌드를 깎는" 재미가 없음

로그라이크와의 충돌:
- PoE = 영속 캐릭터 → 수십 시간 빌드 다듬기 가능
- 로그라이크 = 짧은 런, 죽으면 처음부터 → 빌드 반복 시간 부족
- 이 둘은 본질적으로 충돌

## 논의 1: 로그라이크를 없애면?

사용자 질문:
> "만약 로그라이크라는 개념을 없애면 어떻게 될까?"

### 후보 방안 3개

**방안 A: 로그라이크 제거 → 영속 실험체 RPG (PoE 방향)**
```
연구실에서 실험체 육성
  → 코돈 장착 + 발현 트리 투자 + 연구실 장비
  → 전투 (PvP/PvE)
  → 패배해도 실험체 유지
  → 돌아와서 빌드 조정
  → 다시 전투
  → 교배 = 더 강한 실험체 만들기 (리스타트가 아님)
```
- 장점: 빌드 깊이 문제 자연 해결. "내 실험체"에 대한 애착. 교배 = 새 실험체 추가 (팀 확장). PoE식 "빌드 찾아가는 재미" 그대로 가능
- 단점: "한 판만 더" 중독성 약화. 게임이 길어짐 (3~5분 → ?). 앱인토스 미니앱에 적합한지 의문. 초반 진입장벽 상승

**방안 B: 로그라이크 유지 → 런 내 빌드 압축 (StS/Balatro 방향)**
```
런 시작 → 초기 코돈 3~4개
  → 전투 → 승리 보상에서 선택 (코돈/촉매/발현 포인트)
  → 상점/이벤트에서 빌드 조정
  → 전투 → 보상 → 조정 → ...
  → 패배 → 교배풀 등록 → 재시작
```
- 장점: 짧은 런 유지. 매 런마다 새로운 빌드 실험. 중독성 높음. 앱인토스에 적합
- 단점: 빌드 깊이에 한계 (한 런에 4축 전부 경험하기 어려움). PoE의 "장기 빌드 다듬기"와는 다른 경험. 런이 짧을수록 선택지도 적어짐

**방안 C: 하이브리드 → 영속 연구실 + 짧은 출격 (Hades 방향)**
```
연구실 (영속 공간)
  → 실험체 빌드: 코돈 + 발현 트리 + 장비 장착
  → 출격 (전투 시퀀스, 3~7전투)
  → 전투 중: 런 내 성장 (스킬 흡수, 임시 버프)
  → 패배/완주 → 연구실 복귀
  → 빌드 조정: 코돈 교체, 트리 리스펙, 장비 변경
  → 교배: 새 실험체 생성 (기존 실험체도 유지)
  → 다시 출격
```
- 장점: PoE의 "마을↔맵" 루프 = "연구실↔출격" 루프. 짧은 출격(3~5분)은 유지하면서 빌드 반복 가능. 실험체에 애착 + 교배로 새 실험체도 만듦. 패배 = 리셋이 아니라 "데이터 수집" → 빌드 개선 동기
- 단점: "로그라이크"라는 이름은 빠지지만 출격 구조는 유사. 영속 실험체의 성장 한계를 어디서 잡느냐가 밸런스 과제. 교배풀 시스템 재설계 필요

### 제안 의견

> 방안 C가 가장 유력. 이유:
> 1. 유저가 원하는 "빌드를 깎는 재미" = 연구실에서 조정 → 출격 → 복귀 → 재조정 루프로 해결
> 2. 짧은 플레이 세션(3~5분 출격)은 앱인토스에 적합
> 3. 교배 시스템이 "패배 후 재시작"이 아니라 "새 실험체 추가"가 되면서 수집 요소와 자연 연결
> 4. Hades가 이 구조로 로그라이크 + 영속 진행 양쪽을 성공적으로 잡은 선례가 있음
> 핵심 변경점: 패배 = 실험체 사망/리셋이 아니라, 연구실로 복귀. 실험체는 유지되고, 플레이어가 빌드를 조정해서 다시 출격.

### 결정

사용자: **방안 C 선택**. 성장 한계 도입 필요성 동의.

## 논의 2: 성장 한계 — Mewgenics 참고

사용자 질문:
> "C로 가고 성장한계를 도입해야할 것 같은데 mewgenics에는 어떻게 이 문제를 해결하고 있어?"

### Mewgenics 분석 (2026.02.10 출시, Edmund McMillen)

코어 루프:
```
집 (영속 공간)
  → 고양이 교배/관리 (가구 배치로 교배 품질 조절)
  → 4마리 파티 편성 → 출격 (턴제 전투, StS식 맵 노드)
  → 보스 클리어 → 귀환
  → ★ 출격한 고양이는 "은퇴" → 다시 전투 불가
  → 은퇴 고양이로 교배 → 새 키튼
  → 반복
```

성장 한계 3가지 장치:

1. **스탯 캡 (Base stat max = 7)**
   - 기본 스탯 최대치가 7로 고정
   - 5 = 평균, 6 = 좋음, 7 = 최대
   - 7을 넘기려면 변이(Mutation)가 필요 → 변이는 리스크 동반

2. **은퇴 시스템 (핵심)**
   - 출격 완료한 고양이는 무조건 은퇴 → 다시 전투 못함
   - 아무리 완벽한 고양이를 만들어도 한 번 쓰면 끝
   - 이것이 교배 루프를 강제하는 핵심 장치

3. **영구 사망**
   - 전투 중 전멸 = 영구 사망 (고양이 + 획득 아이템 전부 손실)
   - 전투 외에서도 스트레스, 과밀, 관리 실패로 사망 가능

집 관리 = 빌드 시스템:
- 방마다 5가지 스탯 (Comfort/Stimulation/Health/Mutation/Appeal)
- 가구 배치로 조절 → PoE의 패시브 트리에 해당

### 우리 게임에 적용 — 성장 한계 후보

| 장치 | Mewgenics 방식 | 우리 적용 | 비고 |
|------|---------------|----------|------|
| 스탯 캡 | Base stat max 7 | 코돈 슬롯 수 제한, 스탯 상한 | 단순하고 명확 |
| 은퇴/소모 | 1회 출격 후 은퇴 | 출격마다 실험체 열화 or N회 후 수용 | 교배 강제 장치 |
| 영구 사망 | 전멸 시 손실 | 패배 시 실험체 사망 | 긴장감 |
| 변이 리스크 | 캡 초과는 변이로만 | 동일 적용 가능 | 이미 설계에 변이 있음 |

**A. Mewgenics식 1회 소모**: 한 번 출격하면 은퇴. 교배 재료로만 사용 가능
- 장점: 교배 강제, 깔끔
- 단점: 빌드를 다듬을 시간이 없음 (1회밖에 못 쓰니까). 방안 C의 "빌드 반복" 루프와 충돌

**B. 열화(Degradation)식**: 출격할수록 실험체가 약해짐. N회 후 자연 은퇴
- 장점: 몇 번은 빌드 조정 후 재출격 가능 → "빌드 깎는 재미" 유지. 언제 수용할지 판단이 전략
- 단점: 횟수 밸런싱 어려움

**C. 세대 교체 강제**: 일정 전투 수 이후 교배 필수. 교배하면 새 세대
- 장점: 기존 교배 사이클 설계와 유사. 세대 카운터가 명확
- 단점: "왜 교배해야 하지?"에 대한 인게임 동기 필요

### 제안 의견

> B (열화)가 방안 C와 가장 잘 맞음. 출격할수록 열화 → 빌드를 다듬되 영원히는 못 씀 → 적절한 시점에 교배 → 새 실험체로 이어감.

### 결정

사용자: **B+C 혼합** (열화 + 세대 교체 강제). 밸런싱은 추후.

## 논의 3: 코돈 유전/보관

사용자 질문:
> "B+C로 간다고 했을때, 은퇴하거나 교배되면 보유하고 있던 gene은 다 사라지는거야?"

### 후보 방안 3개

**방안 A: 전부 소멸 — 실험체와 함께 사라짐**
```
실험체 은퇴/교배
  → 보유 코돈(스킬) 전부 소멸
  → 새 실험체는 처음부터 새 코돈 획득
  → 유전되는 건 스탯/속성 경향만
```
- 장점: 깔끔. 매번 새로운 빌드 강제. 밸런싱 단순
- 단점: 빌드 축적감 없음. "내가 모은 게 다 날아간다" → 상실감. 방안 C(영속 연구실)의 취지와 충돌

**방안 B: 연구실 라이브러리 — 코돈이 연구실에 축적**
```
실험체 은퇴/교배
  → 보유 코돈이 연구실 "유전자 뱅크"로 반환
  → 새 실험체에 저장된 코돈 장착 가능
  → PoE처럼: 젬은 캐릭터가 아니라 계정에 귀속
```
- 장점: 축적감. "좋은 코돈을 모으는" 수집 재미. 빌드 빨리 복원 가능. 빌드 반복 루프와 잘 맞음
- 단점: 코돈이 계속 쌓이면 게임이 쉬워짐 → 소모/제한 장치 필요. 새 빌드 시도 동기 약화

**방안 C: 유전적 상속 — 교배를 통해서만 전달**
```
실험체 은퇴/교배
  → 코돈 자체는 소멸
  → 단, 교배 시 부모의 코돈 중 일부가 자식에게 "유전"
  → 유전 확률은 코돈 종류/아미노산에 따라 다름
  → 은퇴(교배 없이) = 코돈 완전 소실
```
- 장점: 생물학 테마와 완벽 일치. 교배의 전략적 가치 극대화. "언제 교배할지" 판단이 핵심 게임플레이
- 단점: 원하는 코돈을 유지하기 어려움 → 유저 좌절 가능. 유전 확률 밸런싱 복잡

### 제안 의견

> B+C 혼합이 가장 좋음:
> - 교배 시: 부모 코돈 일부가 자식에게 유전 (방안 C)
> - 은퇴 시: 코돈이 연구실 유전자 뱅크에 저장 (방안 B), 단 열화된 상태 → 재합성 필요
> - 교배 = 최선의 코돈 전달, 은퇴 = 차선 (비용 필요), 사망 = 전부 손실 (최악)

### 결정

사용자: **B+C 혼합 선택**. 밸런싱은 추후.

## 최종 확정 사항

1. **게임 구조**: 하이브리드 (영속 연구실 + 짧은 출격). 로그라이크 제거. Hades 방향.
2. **빌드 깊이**: 다층 빌드 시스템 (코돈 + 촉매 + 발현 트리 + 연구실 장비). 세부 미설계.
3. **성장 한계**: 열화(B) + 세대 교체 강제(C). 수치 미정.
4. **코돈 유전/보관**: 교배 시 유전(C) + 은퇴 시 뱅크 저장(B). 밸런싱 미정.
