# 논의 004: ATGC 역할 정의

## 배경

구 dawchkins: A=Attack, T=Tough, G=Growth, C=Counter
폐기 이유: "왜 A는 공격이고 T는 방어야?" — 생물학적 근거 없는 자의적 매핑

새로 정의하려면 "왜 이 역할인지" 근거가 필요.

## 후보 3개

### 방안 A: 4스탯 1:1 매핑

```
A → 힘 계열 (공격/체력)
T → 내성 계열 (방어/저항)
G → 민첩 계열 (속도/회피)
C → 변이 계열 (크리/변이 트리거)
```

- 장점: 직관적. 빌드 방향 명확. 하위 gene 설계 쉬움
- 단점: dawchkins과 본질적으로 같음. 생물학적 근거 여전히 없음

### 방안 B: 염기쌍 2축 시스템

생물학에서 A-T가 짝, G-C가 짝 (base pairing). 이걸 게임에 반영:

```
A-T 축 = 전투 축 (뭘 하느냐)
  A = 파괴 (데미지, 관통, 약화)
  T = 보존 (방어, 회복, 보호)

G-C 축 = 변화 축 (어떻게 하느냐)
  G = 질서 (속도, 정밀, 안정)
  C = 혼돈 (변이, 크리, 교란)
```

- 장점: 생물학적 근거 (A-T 쌍, G-C 쌍). 2축이라 조합이 의미를 가짐. 코돈 3글자가 2축 위 좌표가 됨
- 단점: 방안 A보다 이해하기 어려움

### 방안 C: 염기는 역할 없음, 코돈만 의미 있음

- 장점: 가장 자유로움. 자의적 매핑 문제 자체가 사라짐
- 단점: ATGC의 정체성이 없어짐. 하위 gene 차이 불명확

## 결정: 방안 B

사용자: B 선택. 단 네이밍 조정 필요.

### 네이밍 논의

초기: 파괴/보존/질서/혼돈

사용자:
> "B가 좋은데 질서와 보존이 헷갈리는데"

→ "보존"과 "질서" 둘 다 "안전한/지키는" 느낌이라 구분 안 됨

대안 제시:

| | 기존 | 후보 1 | 후보 2 | 후보 3 |
|---|------|--------|--------|--------|
| A | 파괴 | 파괴 | 공격 | 분해 |
| T | 보존 | 생존 | 방어 | 합성 |
| G | 질서 | 가속 | 제어 | 촉진 |
| C | 혼돈 | 변이 | 왜곡 | 변이 |

사용자:
> "파괴, 생존, 질서, 혼돈은 어떄"

→ 축의 의미("어떻게")를 알면 생존과 질서가 안 헷갈림

## 최종 확정

```
     파괴(A)
       |
질서(G)—+—혼돈(C)
       |
     생존(T)
```

| 염기 | 역할 | 축 |
|------|------|-----|
| A (아데닌) | 파괴 | A-T (뭘 하느냐) |
| T (티민) | 생존 | A-T (뭘 하느냐) |
| G (구아닌) | 질서 | G-C (어떻게 하느냐) |
| C (시토신) | 혼돈 | G-C (어떻게 하느냐) |

- A-T 축 = 뭘 하느냐 (파괴 vs 생존)
- G-C 축 = 어떻게 하느냐 (질서 vs 혼돈)
- 코돈 성격 예시: A+C 많음 = "파괴적+혼돈스러운", T+G 많음 = "생존+질서"
